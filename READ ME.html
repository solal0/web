Skira's Blob Factory

Details/Notes

Skira's Blob Factory is an online tool made by @CaptainSkira on WebSim.com
With the power of Websim's blob urls the tool allows anyone to upload a file and get a link to it(1).
The tool's purpose is to give users a simple way of sharing files easily from one end to another.
If you are familiar with blob urls you might know that they are temporary which is true... but WebSim.com does not use your average blob url(2), which means imported files can last until someone deletes them manually.
(1) Above ~15Mb a compiler tool is required to download the file. Read Advanced Steps to understand how to import 15Mb+ files and Compiler Tools Details/Notes to know how to download your file.
(2) Normal blob url "blob:https://example.com" / WebSim's blob url "https://example.com/blobs/"

First Steps into the Blob Factory

Here's a step-by-step guide on how to use the factory for yourself (I recommend you following this guide and only after having your first working blob url checking out Avanced Steps):
  1. Make sure you are on Skira's official Blob Factory. (URLs)
  2. You'll now see 5 buttons in front of you. Click Upload a file and select the file of your choice (must be under 10Mb)
  3. The Generate Link button who was grayed out should now become brighter and clickable. Click it.
  4. After few seconds of wait a link will be generated at the bottom of the website. If you see it, you've successfully made your first upload !
  5. Click the blue text to open your imported file in a new page or click the copy icon next to it to copy the url to clipboard. You can now share this url to anyone.

Advanced Steps

Settings options, what do they do ?:
Bypass Blob Max Size -> Changes the original limit of 10Mb to ~15Mb, only usefull if your file is just a bit bigger than 10Mb. Wether this option is checked or not you'll get a single url to your file.
Big Files Handling -> When importing a file larger than 10Mb/15Mb you need to check that setting. It will cut your imported file in as many chunks of 10Mb as needed and provide you a list of urls all containing a piece of your file.
Accelerated Uploads -> Requires Big Files Handling to be checked otherwise it won't work. Checking it will, on file imported and urls starting to generate, create multiple workers to upload ~8 parts at the same time. This option is more than recommended since without it a 200Mb file takes ~6 minutes to upload while if it was used it would only take ~60 seconds.
Make File Private -> Requires Big Files Handling to be checked otherwise it won't work. When checked a textbox will show up, input a password or whatever you want inside. Close the settings popup and click Generate Link. Now each imported part of your file will be "obfuscated".

Advanced step-by-step guide (15Mb+):
  1. Same steps as for First Steps step-by-step-guide but, before uploading your file, open settings (second button from the right or ⚙️ Button) and check Big Files Handling.
  2. proceed by importing your file and clicking Generate Link.
  3. A list of queued parts will show up. Wait until all parts have their url to proceed.
  4. Once all parts have been uploaded you have 2 options (Copy all is broken):
  5. [Option 1]: You click "Copy Golden Link" who will make you copy a link to your clipboard that when opened will trigger the download of your file using the Online Compiler in silent mode (sample)
  6. [Option 2]: You click "Download Golden Ticket" who will make you download a very lightweight file called {full_file_name}.golden_ticket.txt. This file can be given to all three tools to rebuild the original file. (sample)
  7. If you choosed the golden ticket file you can scroll down to see which tool sounds the best to you and how to use them to turn your golden ticket back into the original file.

Compiler Tools

Details/Notes

Windows tool can handle files of any size and is very fast.
Android tool is slower than the two other tools and might be limited above a certain file size (not confirmed).
Web tool works well and is pretty fast but is limited above a certain file size (~10Gb). He is also sometimes super slow, fetching parts 1 by 1 without providing any errors. And some other times he is super fast.

Windows

Step-by-step quide:
note: step 1 to 6 are only required once.
  1. Download the latest (or 3.14.0) version of python at https://www.python.org/downloads/. Do not download python install manager. Instead click just below the button to download the standalone.
  2. Run python-{latest_version}-amd64.exe and in the window check "Use admin privileges when installing py.exe" and "Add python.exe to PATH" then click Install Now.
  3. Press Yes in the next popup and then wait for Python to install.
  4. Once python Setup was successful press close and open a command prompt by typing cmd in your windows search bar and hitting enter.
  5. In the command prompt type or copy and paste "pip install requests" before hitting enter. Now wait for the installation to complete.
  6. Once downloaded, close the command prompt and download the lastest version of Skira's Blob Compiler for Windows (download)
  7. Extract Skira's Blob Compiler.zip and execute/double click compiler.py located inside the folder you extracted.
  8. If an update is required let it do it's job, press enter when prompted to and relaunch compiler.py
  9. Wait for the golden ticket picker to show up, select the golden ticket you want, select slow or fast option (I recommended using fast) and let the tool do his job.
  10. When all parts have been imported the tool will combine them all as the original file inside the Output folder located in the same directory as compiler.py
  11. If your pc doesn't have enough space the script will ask you to free a certain amount to proceed.

Android

Step-by-step quide:
  1. Download the lastest version of Skira's Blob Compiler for Android (download)
  2. Open SkiraBlobCompiler.apk and install the app. If play protect ask to verify the app, proceed.
  3. Then open the app and inside click Select Golden Ticket File.
  4. Select the golden ticket you want and back in the app click Proceed if you have enough free space.
  5. The app will ask you where to save the the file you're about to download, select a folder and click "Save"
  6. Now you can wait while the app will import each part 1 by 1. When all parts have been successfully imported you can find the file where you told the app to save it at.

Web

Step-by-step quide:
  1. Go to Skira Web Compiler (redirect)
  2. Click Choose .txt / .json and select your golden ticket. (you can also click Paste links / JSON and put a link to your golden ticket raw or just the content of your golden ticket.)
  3. Change download speed to Slow or Fast if you want and click Start.
  4. The website will ask where to save your file, give it a location/directory on your pc and click save.
  5. Now you can wait while the compiler will import each part 1 by 1. When all parts have been successfully imported you can find the file where you told the website to save it at.

Extras

Common/Known Issues

Slow Upload/Download -> Bad Internet/Huge File
Web Compiler printing lot of errors/imported parts value going up and down forever -> most likely to be a memory problem. It happens after ~10Gb downloaded because of the memory being full. There is no fix as of now other than downloading only 10Gb or less files with this Compiler.
Estimated Time not matching (way higher/lower) -> The math done to get these estimated values uses the stats given by a pc who has 800-900Mbps of Download speed and 600-700Mbps of Upload speed. The faster your internet is, the faster files Upload and Download will be.

If you have any other issue, please report them via Discord and i'll see what I can do.

Change Logs

[V1] [WINDOWS]: Handle files called "Golden Ticket.json" containing a name variable and a table of parts. Reads all the parts inside the table and import/add them one by one into a single file called whatever the name variable is.
[V1.1] [WINDOWS]: Now handles "{full_file_name}.golden_ticket.json" and "{full_file_name}.golden_ticket.txt". Same ticket content as before but cleaner. An update checker has been added
[V1.2] [WINDOWS]: Added password handling, when adding a password to your imported file on Skira's Blob Factory the password is saved inside the golden ticket, if found by the tool it will use it to decrypt each part.
[V1] [ANDROID]: Android compiler is out ! Same thing as the windows V1.1 but for mobile and with more details like calculated file size, remaining device space and some other stuff. Does not have update check but has the password detection and part decryption.
[V1.3] [WINDOWS]: Huge optimization added, tool now downloads as fast as the web compiler but can handle 100Gb+ files. Calculated size added next to calculated speed when choosing Slow or Fast method. The tool will make up to 16 workers depending on your pc efficiency to import parts as fast as possible. All imported parts will be added in a folder called ".temp" inside the Output folder and once all parts are imported they will be combined into the original file. Tool requires ~two times the file size you're downloading to import parts and combine them.
